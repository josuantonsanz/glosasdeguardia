Sección del Órgano portativo electrónico

Los órganos portativos, tradicionalmente, tenían un solo registro. Pero el nuestro —por aquello de que es electrónico— se puede permitir tener varios.

Para controlar estos registros —y para algunas funciones extra, que luego veremos— necesitamos algunos botones. Me he decidido por siete pulsadores momentáneos, cada uno con su lucecita LED. Con estos siete registros a mano creo que tendré variedad suficiente para acompañar las eucaristías y las oraciones varias.

Para que esto funcione, vamos a aprovechar el GPIO (que viene a ser la Entrada/Salida de Propósito General) de la OrangePi. Como tenemos siete botones con sus correspondientes luces, necesitamos 14 pines en total (la mitad para entradas y la otra mitad para salidas). Y no nos olvidemos de las resistencias: una para cada LED.

Configuración de los pines

Configuración de los pines de los pulsadores

En este caso, no necesitamos usar resistencias externas, porque los pines del GPIO tienen resistencias internas que podemos controlar por software. En nuestro caso:

  • Vamos a configurar los pines como «pull up»
  • En cada pulsador, conectaremos un terminal al GPIO y otro terminal a tierra

Configuración de los pines de las luces LED

En este caso sí tenemos que usar resistencias. Las conectaremos entre el pin del GPIO y la luz LED, mientras que el otro terminal del LED irá a tierra. Usaremos resistencias de 330Ω (aproximadamente).

De esta manera necesitamos únicamente 15 cables entre los pulsadores y el GPIO: 7 de entrada, 7 de salida y uno a tierra. De hecho, en cada pulsador podemos conectar entre sí los pines de tierra del botón y del LED, y desde ahí sacar un solo cable a tierra.

Instalación del software

Lo primero es instalar python3 y la librería wiringOP. Es importante recordar que los comandos hay que ejecutarlos en root.

sudo apt-get update
sudo apt-get -y install git swig python3-dev python3-setuptools
git clone --recursive https://github.com/orangepi-xunlong/wiringOP-Python -b next
cd wiringOP-Python
git submodule update --init --remote
cd wiringOP-Python
python3 generate-bindings.py > bindings.i
sudo python3 setup.py install

Vamos a poner un ejemplo de script que conecta un botón y una luz, y enciende y apaga el LED conforme se pulsa el botón.

import wiringpi
import time
from wiringpi import GPIO

# Inicializar wiringPi 
wiringpi.wiringPiSetup() 

# Configuración de pines 
LED_PIN = 3 
BUTTON_PIN = 2 

# Configurar LED como salida 
wiringpi.pinMode(LED_PIN, GPIO.OUTPUT) 

# Configurar botón como entrada con pull-up 
wiringpi.pinMode(BUTTON_PIN, GPIO.INPUT) 
wiringpi.pullUpDnControl(BUTTON_PIN, GPIO.PUD_UP)

def main():
    led_state = False  # Estado inicial del LED
    last_button_state = True  # Estado inicial del botón (pull-up = True cuando no está presionado)

    print("Presiona Ctrl+C para salir")

    try:
        while True:
            # Leer el estado actual del botón
            button_state = wiringpi.digitalRead(BUTTON_PIN)

            # Detectar cambio de estado del botón (flanco descendente)
            if button_state <mark> False and last_button_state </mark> True:
                # Cambiar estado del LED
                led_state = not led_state
                wiringpi.digitalWrite(LED_PIN, led_state)
                print(f"LED {'encendido' if led_state else 'apagado'}")

                # Pequeña pausa para evitar rebotes
                time.sleep(0.05)

            last_button_state = button_state
            time.sleep(0.01)  # Pequeña pausa para no saturar la CPU

    except KeyboardInterrupt:
        print("\nPrograma terminado")
        # Apagar LED antes de salir
        wiringpi.digitalWrite(LED_PIN, False)

if __name__ == "__main__":
    main()
