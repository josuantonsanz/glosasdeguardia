Gestor de tareas

4 min read

Objetivo

Crear una aplicación de consola para gestionar una lista de tareas pendientes. Este ejercicio es un pequeño proyecto que te permitirá combinar todo lo que has aprendido: variables, bucles while y for, listas y sus métodos, y condicionales if/elif/else para crear un programa interactivo y útil.

Requisitos

  1. Bucle principal: El programa debe ejecutarse continuamente en un bucle while True, mostrando la lista de tareas y el menú de opciones en cada iteración, hasta que el usuario decida salir.
  2. Mostrar tareas: Al inicio de cada iteración del bucle, el programa debe mostrar la lista actual de tareas.
    • Si no hay tareas, debe mostrar un mensaje como "No tienes tareas pendientes.".
    • Si hay tareas, debe mostrarlas numeradas para que el usuario pueda identificarlas fácilmente (pista: enumerate es perfecto para esto).
  3. Menú de opciones: Debajo de la lista de tareas, debe aparecer un menú claro con las acciones que el usuario puede realizar.
  4. Añadir tarea (opción 1): Si el usuario elige esta opción, el programa le pedirá que escriba la descripción de la nueva tarea y la añadirá al final de la lista.
  5. Completar tarea (opción 2):
    • Si el usuario elige esta opción, el programa le pedirá el número de la tarea que desea completar.
    • El programa debe eliminar la tarea correspondiente de la lista.
    • Importante: Debes controlar posibles errores. ¿Qué pasa si el usuario introduce un número que no corresponde a ninguna tarea (por ejemplo, el número 5 si solo hay 3 tareas)? El programa no debe fallar, sino mostrar un mensaje de error amigable.
  6. Salir (opción 3): Si el usuario elige esta opción, el programa debe mostrar un mensaje de despedida y terminar su ejecución (pista: break).

Pistas y consejos

  • Estructura del programa:
    • Empieza creando una lista vacía al principio de tu script: tareas = [].
    • Usa un while True: para el bucle principal.
    • Dentro del bucle, usa if/elif/else para gestionar la opción elegida por el usuario.
  • Mostrar la lista:
    • Usa un if not tareas: para comprobar si la lista está vacía.
    • Usa un bucle for con enumerate(tareas, start=1) para mostrar las tareas numeradas empezando desde el 1.
  • Completar una tarea:
    • Pide al usuario el número de la tarea y guárdalo como texto.
    • Usa el método de cadena .isdigit() para comprobar si la entrada del usuario es un número.
    • Si es un número, conviértelo a entero con int().
    • Recuerda que los usuarios ven los números desde 1, pero los índices de las listas empiezan en 0. Tendrás que restar 1 para obtener el índice correcto.
    • Antes de intentar borrar, comprueba si el índice es válido (es decir, si está dentro del rango de la lista).
    • El método .pop(indice) es perfecto para eliminar un elemento por su índice. Además, devuelve el elemento eliminado, ¡lo que puedes usar para mostrar un mensaje de confirmación!
  • Limpieza de pantalla (Opcional pero recomendado):
    • Para que la consola no se llene de texto, puedes "limpiar" la pantalla al inicio de cada iteración del bucle. Esto es un extra, pero hace que la aplicación parezca mucho más profesional.
    • Añade al comienzo del código from IPython.display import clear_output y al comienzo del bucle escribe clear_output(wait=True)

Retos extra (si te sientes con confianza):

  1. Evitar duplicados: Antes de añadir una nueva tarea, comprueba si ya existe en la lista para no tener tareas repetidas.
  2. Limpiar toda la lista: Añade una opción "4. Eliminar todas las tareas" que vacíe la lista por completo.
  3. Contador de tareas: Muestra siempre el número total de tareas pendientes en el encabezado. Por ejemplo: --- GESTOR DE TAREAS (3 pendientes) ---.
  4. Tareas completadas: Puedes guardar un contador que diga cuántas tareas has completado ya. Y puedes también guardar otra lista con las tareas completadas, de forma que se puedan ver debajo de las tareas que faltan por completar.
